# Deployment Audit Report

## Scope

Audit of frontend API configuration and deployment/runtime settings with
production target:

- Frontend: <https://pfe-maintenace-industrielle.vercel.app>
- Backend: <https://pfe-maintenaceindustrielle.onrender.com>

## 1. API Base URL Resolution

- Frontend API client resolves base URL from `NEXT_PUBLIC_API_BASE_URL`.
- If missing, it now falls back to
  `https://pfe-maintenaceindustrielle.onrender.com`.
- File: `frontend/src/services/api.ts`

Effective behavior:

- All `apiService` requests target
  `${NEXT_PUBLIC_API_BASE_URL || https://pfe-maintenaceindustrielle.onrender.com}`
  and endpoint path.
- This removes dependency on local proxy paths for production.

## 2. Frontend Environment Variables Required

Set in Vercel (Production and Preview as needed):

- `NEXT_PUBLIC_API_BASE_URL=https://pfe-maintenaceindustrielle.onrender.com`

Template added:

- `frontend/.env.example`

## 3. Backend Environment Variables Required

Minimum required by backend runtime validation:

- `MONGODB_URI`
- `JWT_SECRET`
- `JWT_REFRESH_SECRET`
- `JWT_EXPIRES_IN` or `JWT_ACCESS_EXPIRES_IN`
- `JWT_REFRESH_EXPIRES_IN`

Deployment-critical additions/normalization:

- `CORS_ORIGINS=http://localhost:3000,http://localhost:3001,`
  `https://pfe-maintenace-industrielle.vercel.app`
- `APP_URL=https://pfe-maintenace-industrielle.vercel.app`

Template updated:

- `.env.example`
